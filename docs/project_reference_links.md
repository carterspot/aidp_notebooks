### 🥉 Bronze – Raw Ingestion

```python
# ============================================================
# BRONZE LAYER – Raw Ingestion
# Purpose: Land raw source data into object storage as-is,
#          partition by ingest date, register in catalog.
# ============================================================

from pyspark.sql import SparkSession
from pyspark.sql.functions import current_timestamp, lit
from datetime import date

spark = SparkSession.builder.appName("bronze_ingestion").getOrCreate()

# --- CONFIG -------------------------------------------------
SOURCE_PATH    = "oci://your-bucket@namespace/raw/your_file.csv"
BRONZE_CATALOG = "your_catalog"
BRONZE_SCHEMA  = "bronze"
BRONZE_TABLE   = "your_table_name"
INGEST_DATE    = str(date.today())   # e.g. "2026-03-11"
# ------------------------------------------------------------

# Read raw source (adjust format/options as needed)
df_raw = (spark.read
    .format("csv")
    .option("header", True)
    .option("inferSchema", True)
    .load(SOURCE_PATH))

# Add audit columns
df_bronze = (df_raw
    .withColumn("_ingest_timestamp", current_timestamp())
    .withColumn("_ingest_date", lit(INGEST_DATE))
    .withColumn("_source_file", lit(SOURCE_PATH)))

# Write to Bronze catalog as Delta table (append)
(df_bronze.write
    .format("delta")
    .mode("append")
    .partitionBy("_ingest_date")
    .saveAsTable(f"{BRONZE_CATALOG}.{BRONZE_SCHEMA}.{BRONZE_TABLE}"))

print(f"✅ Bronze ingestion complete: {df_bronze.count()} rows written.")
spark.sql(f"SELECT * FROM {BRONZE_CATALOG}.{BRONZE_SCHEMA}.{BRONZE_TABLE} LIMIT 10").show()
```

### 🥈 Silver – Transformation & Validation

```python
# ============================================================
# SILVER LAYER – Transformation & Validation
# Purpose: Cleanse, deduplicate, validate, and optionally
#          enrich data with GenAI augmentation.
# ============================================================

from pyspark.sql import SparkSession
from pyspark.sql.functions import (
    col, trim, upper, when, current_timestamp,
    row_number
)
from pyspark.sql.window import Window

spark = SparkSession.builder.appName("silver_transform").getOrCreate()

# --- CONFIG -------------------------------------------------
BRONZE_CATALOG  = "your_catalog"
BRONZE_SCHEMA   = "bronze"
BRONZE_TABLE    = "your_table_name"

SILVER_CATALOG  = "your_catalog"
SILVER_SCHEMA   = "silver"
SILVER_TABLE    = "your_table_name_clean"

DEDUP_KEY       = "id"          # Primary key column for deduplication
PARTITION_COL   = "_ingest_date"
# ------------------------------------------------------------

df_bronze = spark.table(f"{BRONZE_CATALOG}.{BRONZE_SCHEMA}.{BRONZE_TABLE}")

# --- 1. Cleanse ---------------------------------------------
df_clean = (df_bronze
    .withColumn("name", trim(upper(col("name"))))   # adjust to your columns
    .filter(col(DEDUP_KEY).isNotNull()))

# --- 2. Deduplicate (keep latest by ingest timestamp) -------
window = Window.partitionBy(DEDUP_KEY).orderBy(col("_ingest_timestamp").desc())
df_dedup = (df_clean
    .withColumn("_row_num", row_number().over(window))
    .filter(col("_row_num") == 1)
    .drop("_row_num"))

# --- 3. Validate (flag bad rows) ----------------------------
df_validated = df_dedup.withColumn(
    "_quality_flag",
    when(col("name").isNull(), "MISSING_NAME")    # extend with your rules
    .otherwise("OK"))

# --- 4. (Optional) GenAI Augmentation -----------------------
# Example: call OCI GenAI or a registered model to add a
# sentiment / classification column. Uncomment to activate.
#
# from pyspark.sql.functions import udf
# from pyspark.sql.types import StringType
# import oci
#
# def call_genai(text):
#     # Insert OCI GenAI API call here
#     return "POSITIVE"
#
# genai_udf = udf(call_genai, StringType())
# df_validated = df_validated.withColumn("_sentiment", genai_udf(col("description")))

# --- 5. Write Silver ----------------------------------------
df_silver = df_validated.withColumn("_silver_timestamp", current_timestamp())

(df_silver.write
    .format("delta")
    .mode("overwrite")
    .option("overwriteSchema", "true")
    .partitionBy(PARTITION_COL)
    .saveAsTable(f"{SILVER_CATALOG}.{SILVER_SCHEMA}.{SILVER_TABLE}"))

good = df_silver.filter(col("_quality_flag") == "OK").count()
bad  = df_silver.filter(col("_quality_flag") != "OK").count()
print(f"✅ Silver complete. Good rows: {good} | Flagged rows: {bad}")
```

### 🥇 Gold – Curation & Analytics-Ready Output

```python
# ============================================================
# GOLD LAYER – Curated, Analytics-Ready Dataset
# Purpose: Aggregate and shape Silver data into trusted,
#          business-facing tables consumable by OAC/FDI.
# ============================================================

from pyspark.sql import SparkSession
from pyspark.sql.functions import (
    col, sum as _sum, avg, count, max as _max,
    current_timestamp
)

spark = SparkSession.builder.appName("gold_curation").getOrCreate()

# --- CONFIG -------------------------------------------------
SILVER_CATALOG  = "your_catalog"
SILVER_SCHEMA   = "silver"
SILVER_TABLE    = "your_table_name_clean"

GOLD_CATALOG    = "your_catalog"
GOLD_SCHEMA     = "gold"
GOLD_TABLE      = "your_summary_table"

GROUP_BY_COL    = "category"   # Adjust to your dimension column
METRIC_COL      = "amount"     # Adjust to your measure column
# ------------------------------------------------------------

df_silver = spark.table(f"{SILVER_CATALOG}.{SILVER_SCHEMA}.{SILVER_TABLE}")

# --- 1. Filter to quality-passed rows only ------------------
df_clean = df_silver.filter(col("_quality_flag") == "OK")

# --- 2. Aggregate -------------------------------------------
df_gold = (df_clean
    .groupBy(GROUP_BY_COL)
    .agg(
        count("*").alias("record_count"),
        _sum(METRIC_COL).alias("total_amount"),
        avg(METRIC_COL).alias("avg_amount"),
        _max(METRIC_COL).alias("max_amount")
    )
    .withColumn("_gold_timestamp", current_timestamp()))

# --- 3. Write Gold (full overwrite — trusted snapshot) ------
(df_gold.write
    .format("delta")
    .mode("overwrite")
    .option("overwriteSchema", "true")
    .saveAsTable(f"{GOLD_CATALOG}.{GOLD_SCHEMA}.{GOLD_TABLE}"))

print(f"✅ Gold curation complete: {df_gold.count()} summary rows.")
spark.sql(f"SELECT * FROM {GOLD_CATALOG}.{GOLD_SCHEMA}.{GOLD_TABLE}").show()

# --- 4. (Optional) Register for OAC consumption -------------
# Grant read access to OAC service principal or data share:
# spark.sql(f"GRANT SELECT ON TABLE {GOLD_CATALOG}.{GOLD_SCHEMA}.{GOLD_TABLE} TO `oac-service-user`")
```

### Session 1 – Getting Started: Provisioning, Catalog & Data Lake

**Core Concepts**
- AIDP = single pane of glass over disconnected data stores, AI models, compute, and governance
- **AIDP** refers to the full ecosystem (ADW, OAC, GenAI, compute, pipelines); **AIDP Workbench** is the UI where you interact with it all
- One **Master Catalog** per instance — the central hub for all data assets (structured + unstructured)

**Catalog Structure**
- **External Catalog** — connection to an Oracle Autonomous Database (no data movement; metadata sync only). Use for ADW, 26ai, FDI-associated databases
- **Standard Catalog** — for file uploads or object storage access. Contains Schemas → Tables + Volumes
  - **Managed Table/Volume**: data lives inside AIDP (backed by OCI Object Storage bucket). Deleting it deletes the data
  - **External Table/Volume**: data stays in your OCI Object Storage bucket. Deleting it only removes the metadata — source data is untouched
- 3-part naming convention: `catalog_name.schema.table`

**Instance Provisioning Tips**
- Choose region carefully: must have OCI GenAI services AND your FDI instance available in same region
- Never use root compartment — create a dedicated compartment for resource isolation and policy enforcement
- Required IAM policies are generated by the UI; optional policies (e.g. object storage access) can be added later
- Tagging is useful when managing multiple AIDP instances (tracking, budgeting)
- Instance creation takes ~10–15 minutes; cluster creation takes ~10 minutes

**Demo Scenario**
- Use case: Supplier data analysis — fragmented multi-source data → governed Medallion pipeline → Gold layer for executive dashboards
- Data sources: supplier basic info (CSV), supplier feedback (CSV), emotion/rating data

### Session 2 – Transform, Curate & Analyze: Workspaces & Medallion Architecture

**Workspaces**
- Workspaces organize projects (notebooks, files, folders) — separate from the Master Catalog which holds data assets
- Compute clusters and workflows are **scoped to a workspace** — not shared across workspaces
- Default workspace + master catalog compute cluster are created automatically on instance provisioning
- Use role-based access within workspaces to control folder/asset visibility per team
- Store config files, wallet files, requirements.txt here (not in catalog)

**Compute Clusters**
- Spark-based execution environment for notebooks and scheduled jobs
- CPU and GPU shapes available; auto-scaling supported
- Attach `requirements.txt` to a cluster via Compute → Library to install Python libraries
- Must attach a cluster to a notebook before running code

**Medallion Architecture — Demo Walkthrough (Supplier Data)**

*Bronze Layer (1 notebook)*
- Programmatically create catalog + schema using Spark SQL (`CREATE CATALOG IF NOT EXISTS`, `CREATE SCHEMA IF NOT EXISTS`)
- Read CSV → DataFrame → write to bronze Delta table with `.mode("overwrite")`
- Can display data via: Spark SQL cell, PySpark `spark.sql()`, or `df.show()`

*Silver Layer (5 notebooks in sequence — order matters)*
1. Create silver catalog/schema; read from bronze; rename columns; add `ingestion_date` via `current_timestamp()`
2. Add `continent` column (nullable); insert bronze data with `NULL` as continent
3. Use OCI GenAI (Gemini 2.5 Pro via `SELECT` against default catalog AI models) to populate continent from country — `ALTER TABLE` to add feedback column
4. Ingest feedback CSV → bronze feedback table; write to bronze catalog
5. Use GenAI to summarize lengthy `REVIEWTEXT_1` into 2 sentences + generate sentiment rating → insert into `silver_supplier_feedback_ai`

*Gold Layer (1 notebook)*
- JOIN silver supplier info + silver feedback AI → `gold_supplier_feedback_ai_join`
- Rename columns for clarity (e.g. `supplier_name_silver` → `supplier_name`)
- Write Gold output to **two external catalogs**: Oracle AI Database 26ai + FDI-associated database (via OX user)
- Verify with `spark.sql("SELECT ...").show(vertical=True)`

**Key Patterns from Demo**
- Use `spark.sql()` for DDL (CREATE CATALOG, CREATE SCHEMA, CREATE TABLE, INSERT INTO)
- Right-click a table in catalog → "Copy Sample Code" to get a pre-built SELECT statement
- GenAI model access via Default Catalog → OCI AI Models (Gemini 2.5 Pro, etc.)
- Query GenAI inline with SQL: `SELECT gemini_model(prompt) AS new_column FROM table`

### Session 3 – Automation & Data Sharing: Workflows, OAC, FDI & Security

**Workflows (Jobs)**
- Workflows = Jobs; made up of a series of **Tasks** that flow through each other
- Task types: **Notebook**, **Python file**, **If/Else** (conditional branching), **Nested Job**
- Tasks can be made dependent: Task 2 only runs if Task 1 succeeds
- **Parameters**: system-defined (run count, run time, day of week) or custom (task-level or job-level)
  - Job-level parameters are visible and editable by all tasks — use for branching logic
  - Task-level parameters are scoped to a single notebook
- **Nested Job tasks** allow entire jobs to be embedded as a single task — ideal for large pipelines broken into Bronze/Silver/Gold sub-jobs
- Scheduling: set from the job's detail page; can also run on demand
- Recommended pattern: create separate Bronze, Silver, Gold jobs → wrap in one Main job using nested job tasks

**Workflow Demo Pattern (Supplier Pipeline)**
```
Main Job
├── Task 1: Bronze (nested job)
├── Task 2: Silver (nested job, depends on Task 1 success)
└── Task 3: Gold (nested job, depends on Task 2 success)
```

**OAC Integration**
- Go to Compute → Cluster → Connection Details → download `config.json`
- In OAC: Create Connection → Oracle AI Data Platform → upload `config.json` + API private key
  - API key fingerprint must match the `config.json`
- Select a specific catalog (bronze/silver/gold) — cannot connect to Master Catalog directly
- Enable "System Connection" if using with a semantic model; otherwise save as dataset connection
- OAC can auto-generate business-friendly column names from technical names using embedded GenAI

**Delta Sharing**
- Create a Share in AIDP → add data assets → define recipients by email
- Recipients do not need AIDP Workbench access — only an email and activation link
- Activation link downloads a JSON → use in OAC as a **Delta Share connection**
- Only tables in **Delta format** are available through Delta Share connections

**FDI Integration**
- Both AIDP and FDI instances must be in the **same OCI region**
- In FDI: Enable Features → enable "Data Share" + "Oracle AI Data Platform" connector
- FDI connection creates a dedicated catalog in AIDP (`fdi_idl_catalog_[region]`)
- Query FDI tables directly from AIDP notebooks via SQL cell using the FDI catalog

**Security & RBAC**
- Two default system roles: **Admin** (full permissions, can grant/revoke) and **Auditor** (read-only audit access)
- Create custom roles (e.g. `data_engineer`) → assign users or other roles to them
- Assign permissions to roles at workspace, catalog, schema, or table level
- **Permissions cascade down**: granting access at workspace level cascades to all contained folders/assets
- Table-level permissions: SELECT, INSERT, UPDATE, DELETE, ALTER
- FDI integration prerequisite: add the FDI OCI ID to the Admin role in AIDP
